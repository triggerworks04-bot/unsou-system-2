# AppSheet 設定メモ（方針のみ）

**本ファイルは AppSheet の画面定義・詳細設定の正本ではありません。**  
列仕様・ENUM・参照関係の正本は **`schema_master.md`** です。

## 現フェーズの方針

- **AppSheet の画面・ビューはまだ作らない。** スキーマ（スプレッドシート列）を先に `schema_master.md` で固める。
- **データソース（Google スプレッドシート）は本フェーズでは直接変更しない。** シート追加・列追加は設計確定後に別作業とする。
- **Regenerate schema** は、スプレッドシート側のシート名・列見出し・列順が **`schema_master.md` と整合したあと**に実施する（下記「Regenerate 前の確認」）。
- **`10_配車予定` は既存列・既存連携（運行予定ID をキーとした点呼・日報・売上等）を壊さない。** 新規は列追加・ENUM 追加による拡張とする。
- **検討案件は `10_配車予定` に直接投入しない。** `11_案件候補` で管理し、決定後のみ `10_配車予定` に反映する。
- **`90_配車表表示用` は参照・印刷・PDF 閲覧用の派生データ想定とし、入力の正本にしない。**
- 新規3シートの **列数** は **`schema_master.md` と一致**させる：**09＝24列**、**11＝33列**、**90＝13列**（**1行目のヘッダー順は同ファイル 節 B-6**）。

## Regenerate schema の前に確認すること（チェックリスト）

1. **シートの新規追加が済んでいるか**（実装タイミング）：`09_定期案件テンプレート`・`11_案件候補`・`90_配車表表示用` はスプレッドシート上に作成後、AppSheet が読み込める状態か。
2. **`10_配車予定` の主キー列**：既存の **運行予定ID** を維持する方針であること（論理名「配車予定ID」との対応は `schema_master.md` の記載どおり）。
3. **列見出し**が `schema_master.md` の「列名」と一致しているか（空白・全角半角・余計な改行がないか）。
4. **ENUM 列**：スプレッドシート上は Text でよいが、AppSheet 側の選択肢とドキュメントの候補値が一致するよう事前にリスト化しているか。
5. **Ref 列**：参照先シートが存在し、参照先の Key 列（運転者ID・車両ID・案件ID・取引先ID 等）が AppSheet で Key として認識されているか。
6. **既存アプリ・既存ビュー**がある環境では、Regenerate による列削除・型変更が発生しないよう、**追加のみ**から始めているか。

詳細なデータモデル・ENUM・説明は **`schema_master.md`** および **`system_design.md`** を参照する。
