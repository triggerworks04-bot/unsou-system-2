# 月別配車表 → 10_配車予定 変換設計

## 1. 目的

この設計書は、**配車予定表_2026年マスター.xlsx** にある **月別横持ち配車表** を、システム連携用の縦持ちDB **10_配車予定** へ変換する**ルール**を定義する。

- **月別配車表**は**人が見やすい**印刷・入力・確認用レイアウトとする。
- **10_配車予定**は**システム連携用**の縦持ちDBであり、Google スプレッドシート上のシート名の例として **10_配車予定** とする。
- **下流システム**（Googleカレンダー、LINE通知、点呼記録簿、運行日報、売上管理表）は **10_配車予定** を**起点データ**として参照・連携する。

列定義の正本は **`docs/schema_master.md` の「B. 配車予定表／10_配車予定」**。本書は**変換の手順・判断ルール**に特化する。

**GAS 実装**（メイン関数・補助関数、入出力シート、複合キーによる UPSERT、初期実装範囲・未決事項など）の詳細は **`docs/dispatch_conversion_gas_spec.md`** を参照する。

---

## 2. 変換対象

- **変換対象は月別配車表シート**である。
  - **例**：`2026年4月`、`2026年5月`、`2026年6月` など、月単位の配車レイアウトが載るシート。
- **対象外**：マスタ、説明、集計、設定などのシート。
- **対象シート名の自動判定ルール**は後続フェーズで確定する。
- **初期段階**では、変換対象として**シート名を手動で指定する**（または設定セルで指定する）方式でよい。

---

## 3. 月別配車表の前提構造

本設計での**前提**。実ファイルと差異がある場合は、本書および `schema_master.md` を先に更新してから変換処理を調整する。

- 月別配車表は**横持ち**形式である。
- **行方向**に**日付**が並ぶ（区切り）。
- **各日付について、原則 2行セット**で構成される。
  - **担当者行**（運転者名を載せる側）
  - **車両行**（車両呼称または車番を載せる側）
- **列方向**に**案件**が並ぶ（見出し列＋複数案件列）。
- **担当者行**の各案件セルには**運転者名**が入る。
- **車両行**の同じ案件列には**車両呼称または車番**が入る。

**例**

| 日付 | 曜日 | 区分 | アート(郡山) | 藤井産業 |
|:---:|:---:|:---:|:---:|:---:|
| 1 | 水 | 担当者 | 村松 | 佐藤 |
| （空または同一日） |  | 車両 | 47-00 | 81-38 |

**配車予定表_2026年マスター.xlsx** の列インデックス・行判定については、次節 **`## 実レイアウト確定事項`** を優先する。

## 実レイアウト確定事項

**配車予定表_2026年マスター.xlsx** の月別配車表は、以下の列構造を前提とする。

| 列 | 項目 | 変換での扱い |
|:---:|:---|:---|
| **A列** | 日付 | **予定日**として使用 |
| **B列** | 曜日 | **曜日**として使用。ただし**予定日を正**とする |
| **C列** | 区分 | **担当者行**／**車両行**の判定に使用 |
| **D列〜「担当者」列の直前** | 案件列 | **`10_配車予定`** への**変換対象** |
| **「担当者」列** | 担当者（見出し） | **変換対象外**。出勤日数確認用のヘッダ列 |
| **「担当者」列の右側** | 出勤日数等 | **変換対象外**。集計・印刷用 |

### 行判定

- **C列が「担当者」** の行を**担当者行**とする。
- **担当者行の直下**で **C列が「車両」** の行を**車両行**とする。
- **担当者行と車両行を 1セット**として読み取る。
- **車両行が見つからない**場合は、その日のデータを**要確認**として扱う。

### 案件列（D列開始〜「担当者」列の直前まで）

- **案件列の開始**は **D列**（列番号固定）。
- **案件列の終了**は **ヘッダー行において「担当者」という見出しの1つ左の列**までとし、**列番号固定では決めない**（案件列は将来 **増減・名称変更**しうる）。
- GAS では **ヘッダー行**から **「担当者」** 列を探し、その**直前まで**を案件列として走査する（`docs/dispatch_conversion_gas_spec.md` の `findHeaderColumnByName_()` / `getLastJobCol_()`）。
- **案件名_入力値** は、各案件列について**見出し行**から取得する。
- **担当者行**の案件セル値を **運転者名_入力値** とする。
- **車両行**の同じ案件セル値を **車両呼称_入力値** とする。

### 対象外列（「担当者」列およびその右側）

- **「担当者」列**（出勤日数確認用のヘッダ列）は、**案件列ではない**。`10_配車予定` の **運転者名**としては**使わない**（運転者名は **案件列内**の担当者行セルから取得する）。
- **「担当者」列の右側**（例：**出勤日数**、その他集計・印刷用）は `10_配車予定` に**変換しない**。

## 4. 10_配車予定の 1行の意味

- **10_配車予定の 1 行は「1日・1案件・1担当者・1車両」** を表す。
- **同じ日に複数案件**がある場合は、**案件ごとに複数行**ができる。
- **同じ運転者が同日に複数案件**を担当する場合も、**案件ごとに複数行**とする。
- **同じ車両が同日に複数案件**に使われる場合も、**案件ごとに複数行**とする。

**例**（論理：`schema_master.md` における **`予定日`** 列に相当する値を「運行日」として示す）

| 運行日（→ `予定日`） | 案件名_入力値 | 運転者名_入力値 | 車両呼称_入力値 |
|---|---|---|---|
| 2026/04/01 | アート(郡山) | 村松 | 47-00 |
| 2026/04/01 | 藤井産業 | 佐藤 | 81-38 |

## 5. 変換ルール

### 5.1 日付の取得

- **担当者行**の **A列「日付」** から、当該行が属する**運行日**を取得する（`## 実レイアウト確定事項`）。
- **車両行**の日付セルが**空欄でも**よい。その場合は、**直近の上位の担当者行**から日付を**引き継ぐ**。
- **年月**は対象シート名（またはブック側の設定）から決まる運用も想定する。初期は「シートが何月か」を**手入力パラメータ**で指定してもよい。
- **日付が取得できない行**については、`10_配車予定` の行を作成せず、ログまたは要一覧で**要確認**とする。

### 5.2 曜日の取得

- **担当者行の B列「曜日」**から取得できる場合は、その値を**初期値**として **`10_配車予定` の `曜日` 列**に書き込む。
- 原則として **`予定日`（§5.1 で確定した運行日を `schema_master` の `予定日` に格納）から自動算出**し、一覧・通知・確認用に用いる。
- **論理データとして正とするのは `予定日`** である。**曜日は表示用**であり、月別表の曜日と `予定日` が不一致の場合は **`予定日` に整合する値へ上書き**する。

**出力方針（`schema_master.md` B. との整合）**：**B. 10_配車予定の `曜日` 列**へ必ず出力する。**曜日は表示用**で、**予定日を正**とする。

### 5.3 案件名の取得

- **D列から「担当者」列の直前まで**を案件列として走査する（`## 実レイアウト確定事項`）。
- 案件列の**見出し（ヘッダ）行**から **案件名_入力値** を取得して保持する（`schema_master.md` に合わせる）。
- **案件マスタ**と照合し、**案件ID** を補完する。
- 案件マスタに存在しない場合は、**マスタ照合状態**を **要確認** または **照合不可** とする運用ルールとする。

### 5.4 運転者名の取得

- **担当者行**の **案件列範囲内**の対象セル値を **運転者名_入力値** として保持する。**「担当者」列（ヘッダ）は運転者名に使用しない**（出勤日数確認用のヘッダ列）。
- **運転者マスタ**と照合し、**運転者ID** を補完する。
- **空欄**の場合は、[5.6 空欄セルの扱い](#56-空欄セルの扱い)に従う。

### 5.5 車両呼称の取得

- **車両行**の同一案件列（**案件列範囲内**）の値を **車両呼称_入力値** として保持する。
- **車両マスタ**と照合し、**車両ID** を補完する。
- **空欄**の場合は、[5.6 空欄セルの扱い](#56-空欄セルの扱い)に従う。
- **対応する車両行**（C列＝「車両」が**担当者行の直下**）が見つからない場合は、当該日ブロックを**要確認**として扱う（`## 実レイアウト確定事項`）。

### 5.6 空欄セルの扱い

| 担当者セル | 車両セル | 変換結果 |
|------------|----------|----------|
| 空欄 | 空欄 | **予定なし**としてスキップ（行を出力しない） |
| 入力あり | 入力あり | `10_配車予定` に出力。**照合OK**または**要確認** |
| 入力あり | 空欄 | `10_配車予定` に出力。**マスタ照合状態＝要確認** |
| 空欄 | 入力あり | `10_配車予定` に出力。**マスタ照合状態＝要確認** |

※「入力あり」のみで業務運用しない案件は運用側で運転者名入力を必須にし、コンバート側は上表に従う。

### 5.7 作成元情報の保存

- **作成元シート名**を保存する（`schema_master.md` の同名列）。
- **作成元セル**を保存する。
- **可能なら**、担当者セルと車両セルの両方を追跡する。
- **初期設計**：**作成元セルは担当者側セルアドレス**を代表値として記録してよい（車両側はログまたは将来拡張）。

### 5.8 ステータスの初期値

- **通常変換時**、`10_配車予定` の **ステータス初期値は「確定」** とする運用が基本だが**現場運用次第で調整可**。
- 「要確認」データは、`ステータス` を増やして表現せず（予定／変更に無理に落とさず）、まず **`マスタ照合状態`** で状態管理する。
- **キャンセル**や**内容変更の履歴管理**は、主に **`10_配車予定` 上で**、`ステータス`・`更新日時` とあわせて行う。

### 5.9 マスタ照合状態

`schema_master.md` の **`マスタ照合状態`** と整合する。**候補**：

| 状態 | 意味 |
|------|------|
| 未照合 | まだマスタ照合を実施していない |
| 照合OK | 案件・運転者・車両がいずれも照合または不要と判定できた |
| 要確認 | 一部照合できない、または規定外の空欄パターン |
| 照合不可 | 必須情報が欠け、運用として確定運行として使わない／再入力が必要 |

## 6. マスタ照合ルール

- **案件名_入力値** → **案件マスタ**
- **運転者名_入力値** → **運転者マスタ**
- **車両呼称_入力値** → **車両マスタ**（車両マスタの **車両呼称** と照合することが多い）

**基本方針**

- **照合は完全一致を基本**とする。
- **将来的**：別名・略称によって照合できるよう、別テーブルを持つ。

**別名検討（将来）**

| 項目 | メモ |
|------|------|
| 案件別名マスタ | 略称ヘッダと正式案件名を橋渡し |
| 運転者別名マスタ | 異体字・愛称など |
| 車両別名マスタ | 車番・社内コードのゆれ対応 |

また、**表記ゆれ**への対処として **`照合エラーメモ`** とあわせ、**別名列**または **エイリアスマスタ** の導入を検討する。

## 7. 10_配車予定へ出力する列

`schema_master.md` の **B.（10_配車予定）** と同一の名前をなるべく用いる。主な出力列：

| 列名 | 出力元 | 備考 |
|------|--------|------|
| 運行予定ID | AppSheet **`UNIQUEID()`**／変換処理で発番 | **内部 Key** |
| **予定日** | **担当者行の A列** | 運行日 → **`予定日` 列**に格納 |
| **曜日** | **担当者行の B列**（初期値）または**予定日からの自動算出** | **`schema_master.md` B. の `曜日` 列**へ出力。**表示用**で**予定日を正**とする |
| 案件名_入力値 | **案件列範囲**の見出し行 | 元表の文字列を保持 |
| 案件ID | 案件マスタ照合 | Ref |
| 運転者名_入力値 | **担当者行**の **案件列**該当セル | **「担当者」ヘッダ列は対象外** |
| 運転者ID | 運転者マスタ照合 | Ref |
| 車両呼称_入力値 | **車両行**の **案件列**該当セル | **「担当者」列より右は対象外** |
| 車両ID | 車両マスタ照合 | Ref |
| ステータス | 初期値設定 | **通常は「確定」**（運用により調整） |
| 出発予定時刻 | **案件マスタ等で補完** | **`Time`**（`calendar_sync_design.md` との整合） |
| 到着予定時刻 | 同上 | **`Time`** |
| 出発地 | **案件マスタ等で補完** | `Text` |
| 目的地 | 同上 | `Text` |
| 作成元シート名 | 対象シート名 | 追跡用 |
| 作成元セル | 担当者セル位置（初期） | 追跡用 |
| マスタ照合状態 | 照合結果 | Enum |
| 照合エラーメモ | 失敗理由 | LongText |

**曜日（注記）**：**`schema_master.md` B.（10_配車予定）の `曜日` 列**へ出力する。**曜日は表示用**であり、**予定日を正**とする。原則として**予定日から自動生成**する。月別配車表から取得できる場合は**初期値**として取り込むが、**最終的には予定日に整合する値**へ揃える（§5.2）。

カレンダー・LINE で必要な **`カレンダーイベントID`・`最終同期日時`・`LINE送信済みフラグ`** などは、`schema_master.md` に従い `10_配車予定` に保持し、変換処理では変更不要な列はそのまま**既存値を維持**する（UPSERT で上書きルール参照）。

## 8. 重複防止ルール

同じ月別配車表ソースから **再実行**されたときに、同じ論理予定が**二重に増えない**こと。

**初期案：複合キー（論理一意）**

- **作成元シート名**
- **予定日**（運行日）
- **案件名_入力値**
- **作成元セル**（初期は担当者セル）

**挙動**

- 上記で **一致する既存の `10_配車予定` 行**があれば、**INSERT ではなく UPDATE** とする。
- **日報／点呼**と既に関連付いた行については、`運行予定ID` は原則**維持**しフィールドのみ更新すること。

**将来**

- **変換バッチID**
- **更新履歴ログ** を持ち、変更の検証やロールバックを可能にする。

---

## 9. 変更・キャンセルの扱い

- 月別配車表を修正して **再変換**した場合、複合キーにより **同一 `運行予定ID` を指す既存行を更新**する（[8](#8-重複防止ルール)）。
- **月別配車表から予定セル組が「消えた」**ときの振る舞い：**未決**。

**候補**

1. **`10_配車予定` の `ステータス` を `キャンセル`** とする。
2. 行を物理**削除**する。

**初期方針（推奨）**

- **削除せず**、**ステータス＝キャンセル** とする。
- **日報・点呼連携済み**の予定については **物理削除しない**。

## 10. 変換実行方式

### 初期段階（想定）

- **手動実行**または**カスタムメニュー／ボタンによる実行**
- **対象月（または対象シート）** を指定して変換
- 結果を **`10_配車予定`** に出力
- **マスタ照合状態＝要確認／照合不可**の行を一覧で確認できるようにする（スプレッドシートまたは AppSheet で）

実装：**将来的には GAS** でスクリプト化する。**最初から完全自動**にはしない前提。

### 将来

- GAS で**定期自動変換**
- 「毎日夜」または「配車確定後」イベントで実行し、**Googleカレンダー同期～LINE通知生成**との連係を強化する

## 11. 下流連携

- **Googleカレンダー連携**：`10_配車予定` を対象（`calendar_sync_design.md`）。
- **LINE通知**：**翌日分**を `10_配車予定` から抽出（`line_notification_design.md`）。
- **点呼記録簿**：`10_配車予定` を運行選択の**候補**として参照。
- **運行日報**：`10_配車予定` を参照し**日報ヘッダ**を起票できるようにする。
- **売上管理**：`10_配車予定` と **日報実績**を参照して計上根拠を辿れるようにする。

## 12. 未決事項

確定済み項目は **`## 実レイアウト確定事項`** に記載。**未決**は運用・実装フェーズで本書および `schema_master.md` を更新する。

### 12.1 確定済み（配車予定表_2026年マスター.xlsx）

| 項目 | 内容 |
|:---|:---|
| **日付列・曜日列・区分列** | **A列＝日付**（予定日）、**B列＝曜日**（表示用／予定日を正）、**C列＝区分**（担当者／車両の行判定） |
| **案件列の開始** | **D列**（固定） |
| **案件列の終了** | **ヘッダー行で「担当者」列の1つ左まで**（**自動判定**。列番号固定にしない） |

### 12.2 未決（要検討）

| 項目 | メモ |
|------|------|
| **対象シート名の判定ルール** | 初期は手動指定。[§2](#2-変換対象) と整合 |
| **月別配車表から消えた予定の扱い** | [§9](#9-変更キャンセルの扱い) と整合 |
| **案件マスタの標準出発時刻・出発地・目的地**の整備 | 変換後補完の前提 |
| **別名／略称マスタ**を作るかどうか | `照合エラーメモ` とセット |
| **変換処理を手動ボタンにするか**、**自動実行にするか** | 初期は手動寄り（[§10](#10-変換実行方式)） |
