# 配車予定表（カレンダー／予定同期）設計

## 1. 位置づけ

運行システムの**システム連携の起点**は **10_配車予定**（`schema_master.md` の「B.」）である。印刷用の **月別配車表（横持ち）からは直接連携しない**。**10_配車予定** で管理する **運行予定ID（内部Key）** をもとにカレンダーを更新し、`system_design.md` の通り下流（点呼・日報・売上・LINE）とも整合させる。

**前提**：**10_配車予定** の行は、`docs/dispatch_conversion_design.md` のルールに従って、月別横持ち配車表を**変換**して生成・更新されたデータであるとする。

## 2. 連携対象：10_配車予定のみ

- **Googleカレンダー連携**は、**印刷用の月別配車表を対象にしない**。**10_配車予定**のみを対象とする。
- 月別横持ち表は**人間用**のままとし、変換後に 10_配車予定 が整ってからカレンダーを触る。

## 3. 保存・入力・列

- **保存先**：Google スプレッドシート上の **`10_配車予定`** シート（名称例）。列定義は `schema_master.md` の「B.」。
- **現場入力**：主に月別表での調整のうえ、**変換処理**で 10_配車予定 を更新。**AppSheet** で 10_配車予定 を直接編集するかは運用で任意。
- **カレンダー連携**：**カレンダーイベントID**・**最終同期日時**・**ステータス** で状態を管理。GAS 等でイベントと 1対1 で対応。

## 4. 運行予定ID と表示番号

- **運行予定ID**: AppSheet **UNIQUEID()**（内部Key）。予定変更時も **原則同一キーを維持**し、**ステータス**と**更新日時**で追う（全体方針は `README.md`／`system_design.md`）。
- **運行予定番号**: 表示用。後続で GAS または式により整備可。

## 5. 同期対象行とイベント内容

### 連携対象

- **10_配車予定**のうち、**ステータスが「確定」**または運用で定めた**連携対象**の行のみ Google カレンダーへ反映する。（「連携対象」を別 Enum に切り出す場合は `schema_master.md` で追記）

### カレンダー表示名（イベントタイトルの基本形）

```
【案件名：車両呼称 運転者名】出発地 出発予定時刻～
```

- **案件名**：原則 **案件名_入力値** またはマスタ照合後の **案件マスタ／Ref から解決した表示名**（運用で統一）。
- **車両呼称・運転者名**：**車両呼称_入力値**／**運転者名_入力値**、またはマスタ照合後の表示。

**例**：

```
【アート(郡山)：47-00 村松】郡山支店 8:00～
```

### 案件マスタからの補完

- **案件マスタ**に **出発地**・**目的地**・**標準出発時刻** 等を持たせ、変換または同期前処理で **10_配車予定** の空欄を**補完**する方針とする（列の追加は `schema_master.md` の案件マスタと B を先に更新）。

## 6. 同期管理の方針（技術）

| 列 | 役割 |
|----|------|
| **カレンダーイベントID** | Google カレンダーのイベントとの**突合キー** |
| **最終同期日時** | 当該行についてカレンダーへ反映した時刻 |
| **ステータス** | 同期対象か、タイトル・時刻にどう反映するかの判断材料 |

**予定時刻の型**：`schema_master.md` に従い、**出発／到着の予定時刻は Time**、**運行日は `予定日`（Date）**。カレンダーの **開始／終了 DateTime** は GAS で **`予定日` ＋ `出発予定時刻`／`到着予定時刻`** を結合して組み立てる。

処理フロー（概念）：

1. 10_配車予定で確定／変更 → イベントID があれば **更新**、なければ **作成**してイベントID を書き戻す。
2. 同期成功ごとに **最終同期日時** を更新。
3. **キャンセル**は下記「未決事項」のいずれかで運用決定後、ステータスとあわせてカレンダー側を整合。

## 7. 他システムとの関係

- **LINE送信済みフラグ**（10_配車予定 側）：翌日通知の重複防止。詳細は `line_notification_design.md`。
- 点呼・日報では **運行予定ID**（10_配車予定 由来）で選択または引き込み。
