# 配車予定表（カレンダー／予定同期）設計

## 1. 位置づけ

配車予定表は **運行システム全体の最上流データ** である。ここで管理する **運行予定ID（内部Key）** を点呼記録簿・運行日報・売上管理表が参照する。**ステータス** は予定変更・キャンセル・**カレンダー同期・LINE通知** の制御に使う。

## 2. 保存・入力

- **保存先**: Google スプレッドシート。列定義は `schema_master.md` の「B. 配車予定表」。
- **現場入力**: AppSheet。
- **カレンダー連携**: **カレンダーイベントID**・**最終同期日時**・**ステータス** で同期状態を管理する。GAS 等でイベント作成／更新／突合するときは、スプレッドシート側のイベントIDとイベントを1対1で対応させる。

## 3. 運行予定ID と表示番号

- **運行予定ID**: AppSheet **UNIQUEID()**（内部Key）。予定変更時も **原則同一キーを維持**し、**ステータス**と**更新日時**で追う（全体方針は `README.md`／`system_design.md`）。
- **運行予定番号**: 表示用（例 `YP-YYYYMMDD-001`）。後続フェーズで GAS または式により整備可。

## 4. 同期管理の方針（カレンダー）

| 列 | 役割 |
|----|------|
| **カレンダーイベントID** | Google カレンダーのイベントとの**突合キー** |
| **最終同期日時** | 当該行についてカレンダーへ反映した時刻 |
| **ステータス** | 同期対象か、タイトル・時刻・説明文にどう反映するかの判断材料 |

処理フロー（概念）：

1. 配車側で確定／変更 → イベントID があれば **更新**、なければ **作成**してイベントID を書き戻す。
2. 同期成功ごとに **最終同期日時** を更新。
3. **キャンセル**は下記「未決事項」のいずれかで運用決定後、ステータスとあわせてカレンダー側を整合させる。

## 5. 他システムとの関係

- **LINE送信済みフラグ**：翌日運行予定などの送信済み可否。詳細は `line_notification_design.md`。
- 点呼・日報では **運行予定ID** で選択または引き込み。
