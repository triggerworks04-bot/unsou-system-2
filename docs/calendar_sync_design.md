# 配車予定表（カレンダー／予定同期）設計

## 1. 位置づけ

配車予定表は **運行システム全体の最上流データ** である。ここで管理する **運行予定ID** を、点呼記録簿・運行日報・売上管理表が参照することで、計画と実績・売上を論理的に接続する。

## 2. 保存・入力

- **保存先**: Google スプレッドシート（1シートまたは専用スプレッドシート）。列定義は `schema_master.md` の「B. 配車予定表」に従う。
- **現場入力**: AppSheet（予定の閲覧・担当者による更新が必要な場合）。
- **カレンダー連携**（任意）: Google カレンダー等との同期は、GAS または外部連携で **運行予定ID を説明欄または拡張プロパティに保持**できるようにすることが望ましい。詳細は実装フェーズで確定する。

## 3. 運行予定ID

- 1行＝原則1予定運行とし、運行予定IDで一意。
- **採番規則**（例）: `YP` + `YYYYMMDD` + `-` + 連番。実際のルールは ChatGPT／現場との合意後、`schema_master.md` の変更履歴に記載する。

## 4. 他システムとの関係

- 点呼・日報では **運行予定ID** をキーとして選択または自動引き込みできるUIをAppSheet側で検討する。
- 予定のみ存在し実績がない期間でも、配車単独運用として成立させる。
