# 点呼記録簿 設計

## 1. 目的

出庫前等の **点呼実施記録** を残し、法令・社内規程に沿った運用を支える。**単独**では点呼記録のみで完結させ、連携時は **運行予定ID・運転者ID・車両ID** で **10_配車予定**／日報と突合可能にする。

## 2. データ項目

論理モデルは `schema_master.md` の「C. 点呼記録簿」を正とする。AppSheetでは次を想定する。

- **点呼ID**（UNIQUEID）／**点呼番号**（表示用・後続）
- 運行予定ID（連携運用では選択または自動）
- 運転者・車両の実施対象（マスタ Ref）
- **点呼種別**（出庫前／中間／帰庫後）、**実施日時**
- **アルコール検知値**、**異常有無フラグ**、**体調確認区分**（良好／要確認／不可）
- **指示事項**（LongText）
- **点呼者ID**（運行管理者マスタ Ref。未整備時は運転者マスタを暫定参照可）

## 3. AppSheet での入力フロー（案）

1. 当日の自分の運行を **運行予定IDまたは日付＋運転者** で絞込
2. 点呼種別ごとに1行追加
3. （任意）運行予定に紐づかない臨時便は運行予定IDを空にし、単独運用として保存

## 4. GAS／自動化の余地

- **10_配車予定**側から同日・同運転者の予定を参照し、候補の運行予定IDを提案する処理（将来）。
- 監査用に、編集ログやタイムスタンプ列をマスタ側で統一する。
