# 点呼記録簿 設計

## 1. 目的

出庫前等の **点呼実施記録** を残し、法令・社内規程に沿った運用を支える。**単独**では点呼記録のみで完結させ、連携時は **運行予定ID・運転者ID・車両ID** で配車／日報と突合可能にする。

## 2. データ項目

論理モデルは `schema_master.md` の「C. 点呼記録簿」を正とする。AppSheetでは次を想定する。

- 運行予定ID（連携運用では選択または自動）
- 運転者・車両の実施対象（マスタから選択推奨）
- 点呼実施時刻、実施区分（出勤前／中間／その他など）
- 記録担当者、アラコメント／異常有無

## 3. AppSheet での入力フロー（案）

1. 当日の自分の運行を **運行予定IDまたは日付＋運転者** で絞込
2. 点呼種別ごとに1行追加
3. （任意）運行予定に紐づかない臨時便は運行予定IDを空にし、単独運用として保存

## 4. GAS／自動化の余地

- 配車側から同日・同運転者の予定を参照し、候補の運行予定IDを提案する処理（将来）。
- 監査用に、編集ログやタイムスタンプ列をマスタ側で統一する。
