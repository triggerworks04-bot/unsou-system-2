# 売上管理表 設計

## 1. 目的

**案件・取引先** を軸に売上を管理する。連携時は **案件ID・取引先ID** に加え、**運行予定ID** および **運行日報ID（日報ヘッダ Ref）** で **10_配車予定**／日報と突合し、**計上根拠**を辿れるようにする。

## 2. データの正

金額・請求月・入金・税区分などの列は `schema_master.md` の「F. 売上管理表」を正とする。マスタは `04_07_共通マスタ雛形` および案件／取引先マスタと整合させる。

## 3. 単独利用と連携

- **単独**: 運行予定ID・運行日報IDを使わず、案件IDのみで運用可能にする余地を残す（ただしID統一を推奨）。
- **連携**: **運行日報ID** で日報ヘッダと一意に結びつけつつ、必要に応じ **運行予定ID** や GAS で集計した数量を参照する。

## 4. 運送日報集計との関係

- 集計シート／フォルダ「運送日報集計」で使っている **キー**（例: 月次、案件、ドライバー別）を洗い出し、売上側のディメンションと一致させる。
