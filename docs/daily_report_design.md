# 運行日報 設計

## 1. 目的

日々の **運行実績**（走行、荷役、休憩、異常など）を記録する。紙／PDF様式は `運送日報フォーマット.pdf`・記入例を参照し、電子化する列は `schema_master.md` の「D. 運行日報」を正とする。

## 2. 単独利用と連携

- **単独**: 運行予定IDなしでも、実走行日＋運転者＋車両で運用可能。
- **連携**: 配車側の **運行予定ID** を入力または参照し、計画との差異分析が可能になる。

## 3. 参照資料とのマッピング

| 参照元 | メモ |
|--------|------|
| 運送日報フォーマット.pdf | 画面上のグルーピング・必須表示の優先順位 |
| 運送日報_記入例.pdf | バリデーション・プレースホルダ説明 |
| 08_11_運送日報入力雛形 | 列の一覧取り込み時の一次ソース |
| 運送日報集計 | 集計キー（日付／案件／車両等）を売上・分析にフィード |

## 4. AppSheet

- 日付・便・運転者での絞込、オフライン考慮は別途検討。
- 入力完了後にスプレッドシートへ即時反映（標準のAppSheet動作）。
