# 配車変換 GAS 実装仕様（月別横持ち → 10_配車予定）

## 1. 目的

この仕様書は、**月別横持ち配車表**から **`10_配車予定`** を生成する **Google Apps Script（GAS）** 実装の指針を定義する。

- **詳細な変換ルール**（列範囲・空欄・行セット・論理）は **`docs/dispatch_conversion_design.md`** を正とする。
- **列定義の正本**は **`docs/schema_master.md`** の「**B. 配車予定表／10_配車予定**」。
- **本書**では、GAS における**関数構成**、**入出力シート**、**UPSERT／更新ポリシー**、および**初期実装のスコープ**を定義する。

## 実装ファイル

リポジトリ上のソース（スプレッドシート側のプロジェクトへコピーして利用する）：

- **`scripts/dispatch_conversion.gs`**
- **第1版**では、月別横持ち配車表から **`10_配車予定`** への**縦持ち変換と UPSERT のみ**を行う。
- **Googleカレンダー連携・LINE通知・マスタ照合の高度化**は**後続フェーズ**で拡張する。

---

## 2. 実装方針

- **初期実装**は **手動実行**（エディタから）または**スプレッドシート上のカスタムメニュー／図形ボタンからの実行**とする。**完全自動**（時間主導トリガのみ）にはしない。
- **対象月の配車シート名**を、**設定値**（例：名前付きレンジ・設定シート・`PropertiesService`）または**メイン関数の引数**で指定して変換する。
- **月別配車表**はそのまま**印刷・入力・確認用**とし、**Googleカレンダー／LINE／点呼／日報／売上**には**直接**つなげない。**下流連携は常に `10_配車予定` を起点**とする。
- **変換結果は `10_配車予定` シートへ書き込む**（同一スプレッドシートを想定。別ブックの場合は API／IMPORTRANGE など別検討）。
- **優先順位**：第1フェーズでは**正しく縦持ち化し、UPSERT で整合する**こと。**マスタ照合の高度化・カレンダー同期・LINEキュー連携は後続フェーズ**で行う。
- **後続フェーズ**では、Google カレンダー同期や LINE 通知キュー生成 GAS と**連係**させる。

### 読み取り元

- **月別配車表**シート（例：`2026年4月 配車表`）。
- **初期実装**：**対象シート名**は**関数引数**または**設定セル／スクリプトプロパティ**で指定する（`dispatch_conversion_design.md` §12 の「対象シート名の判定」は別途）。

| 車両マスタ | 同上 | |
| 案件マスタ | 同上 | 時刻・住所補完も後続 |

## 4. 前提レイアウト

`dispatch_conversion_design.md` の **`## 実レイアウト確定事項`** に従う。

**確定**：案件列は **D列開始**。終了は **ヘッダー名「担当者」列の直前**を **自動判定**（列固定しない）。

| 列 | 内容 | GAS での扱い |
|:---|:---|:---|
| **A列** | 日付 | **予定日**の元（対象年月と結合して `Date` 化） |
| **B列** | 曜日 | 初期値。**`予定日` を正**とし、`getWeekdayText_()` 等で検証／上書き可 |
| **C列** | 区分 | **「担当者」行／「車両」行**の判定 |
| **D列以降（途中まで）** | 案件列 | **ヘッダー行で「担当者」列を探し、その1つ左まで**を横持ち→縦持ちの走査対象とする |
| **「担当者」列** | 担当者（見出し） | **変換対象外**（出勤確認用ヘッダなど） |
| **「担当者」列の右側** | 出勤日数等 | **変換対象外**（集計・印刷用） |

- 「**担当者**」という文字は **C列の行区分**（データ行の担当者行判定）にも存在するため、**案件列終了判定では A〜C 列を検索せず**、**D 列以降から**ヘッダー名「担当者」を探す。
- **C 列の「担当者」**：**行判定用**。**右側ヘッダーの「担当者」**：**列停止位置用**として扱う。

**行・列ルール**

- **C列＝「担当者」**の行を**担当者行**とする（見出しの「担当者」列とは別。C列は**区分**）。
- その**直下**で **C列＝「車両」** の行を**車両行**とする。
- **案件名**：**見出し行**の **D列開始〜「担当者」ヘッダの直前まで**の各列から取得する。
- **案件列の終了位置**：ヘッダー行を **D列（`FIRST_JOB_COL`）以降のみ**走査し、セル文字列（正規化後）が **`担当者`** と一致する最初の列を **停止列** とし、**その左隣列まで**が案件列。**A〜C列には見出し「担当者」が無い前提でも、誤って C列の区分文言を拾わない。** **停止列と右側すべて**は変換対象外（**出勤日数**等を含む）。
- 「**担当者**」という文字は **C列の行区分**（データ行の担当者行／車両行判定）にも現れる。**案件列終了判定では A〜C 列を検索対象にせず**、**D列以降から**ヘッダー名「担当者」を探す。
- **C列の「担当者」**：**行判定用**。**右側ヘッダーの「担当者」**：**列停止位置用**として扱う（`findHeaderColumnByName_` は `startCol` に `FIRST_JOB_COL` を渡す）。
- **見出し行の行番号**は、**初期実装ではスクリプト定数または設定値**（例：`HEADER_ROW = 2`、`START_ROW = 3`）として持つ。**実ファイルに合わせて後続で確定**する。

## 5. 初期実装範囲

### 初期実装で行う

- 対象シートの**読み取り**（存在チェック含む）
- **担当者行＋車両行**の 2 行セット**判定**
- **案件列**（D列〜「担当者」見出しの直前まで）の横持ちセルを **1 行＝1 案件**の縦持ちレコードに**展開**
- **空欄セル**の扱い（§7）
- **`10_配車予定` への追加または更新**（§8）
- **作成元シート名**、**作成元セル**（担当者セル）の保存
- **マスタ照合状態**の**初期値**設定（§9）
- **曜日**の出力（B 列または `予定日` から算出し、**予定日を正**）
- **実行ログ**・**要確認件数**の表示（§12）

### 初期実装で行わない

- **Google カレンダー**への同期
- **LINE** 送信・**LINE通知キュー**の生成
- **点呼記録簿**の自動作成
- **運行日報**の自動作成
- **売上**明細の自動作成
- **高度なマスタ照合**（別名・あいまい一致など）
- **月別表から消えた予定**の**自動キャンセル**（ステータス更新）
- **物理削除**による行の消去

## 8. 重複防止・更新ルール

初期実装では、以下の**複合キー**で既存行を検索する（`dispatch_conversion_design.md` **§8** と同一）。

1. **作成元シート名**
2. **予定日**（年月日のみで正規化）
3. **案件名_入力値**
4. **作成元セル**（担当者セル、アドレス文字列）

**既存行がある場合**

- **`運行予定ID`** は**変更しない**。
- **予定日／曜日／案件名_入力値／運転者名_入力値／車両呼称_入力値／ステータス**（入力由来）などを**上書き**する。
- **`カレンダーイベントID`・`LINE送信済みフラグ`・最終同期日時**など、**後続連携由来の列は原則そのまま**（意図的に変換だけで連携フラグを落とさない）。

**既存行がない場合**

- **新規行を追加**し、**運行予定ID** を新規発番する。
- **ステータス＝確定**（運用により変更可）。
- **`LINE送信済みフラグ`＝FALSE**（または未送信相当）。

**注意**

- **日報・点呼・売上と紐づく行**を**不用意に削除しない**。
- **初期実装では**、月別表から消えたセルに対する**自動キャンセル**（`ステータス＝キャンセル`）は**行わない**。

## 9. マスタ照合状態の初期ルール

**初期実装ではマスタテーブルとの照合は行わない**（ID 列は空のまま）。

**初期値（案）**

| パターン | マスタ照合状態 |
|----------|----------------|
| **担当者セル／車両セルとも入力あり** | **未照合** |
| **どちらか一方のみ入力あり** | **要確認** |
| **両方空欄** | **行を出力しない**（§7） |

**後続フェーズ**

- 案件・運転者・車両マスタと照合し、**照合OK／要確認／照合不可**を付与する。
- `照合エラーメモ` に理由を残す。

## 10. 主な GAS 関数案

| 関数名 | 役割 |
|--------|------|
| `convertDispatchSheetToSchedule()` | **エントリ**。対象シートを `10_配車予定` へ変換する |
| `readDispatchSheet_()` | 月別配車表を **値の二次元配列**等で読む |
| `buildScheduleRows_()` | 担当者行・車両行・案件列から**論理行オブジェクト**の配列を生成 |
| `findExistingScheduleMap_()` | 既存 `10_配車予定` を **複合キー → 行番号 or レコード** にマップ |
| `upsertScheduleRows_()` | 新規追記と既存行の更新 |
| `buildSourceKey_()` | 重複判定キー文字列の生成 |
| `createScheduleId_()` | 新規 **UUID** 等の発番 |
| `getWeekdayText_()` | `予定日` から曜日表示文字を返す |
| `normalizeCellValue_()` | **trim**・空文字統一など |
| `findHeaderColumnByName_()` | ヘッダー行から**指定名の列番号**（1起点）を探す。**`startCol`** 以降のみ検索し、案件終了用の「担当者」で C列区分を誤検出しない |

| `getLastJobCol_()` | **「担当者」** 見出し列の**直前**を案件列の最終列として返す |

内部関数は末尾 `_` の慣習に合わせる（命名は実装時に調整可）。

## 11. 処理フロー

1. **対象シート名**を取得（引数／設定）。
2. 月別配車表を **readDispatchSheet_** で読み込む。
3. **案件ヘッダ行**から **「担当者」** 列を特定し、その**左隣まで**を案件列として **案件名**を取得する（`findHeaderColumnByName_`／`getLastJobCol_`）。
4. データ行を走査し、**C列＝担当者**の行を見つける。
5. **直下の行**が **C列＝車両**か確認する（違う場合は [§12](#12-エラー要確認処理)）。
6. **案件列範囲**を列方向に走査する。
7. **担当者セル・車両セル**の組から **1 論理行**を組み立てる（§7 でフィルタ）。
8. `10_配車予定` の既存データを **findExistingScheduleMap_** でキー化する。
9. キー一致なら **更新**、なければ **追加**（**upsertScheduleRows_**）。
10. **Logger** または**結果シート**にサマリを出力する。
11. **要確認件数**を **UI / toast** で示す（可能なら）。

---

## 12. エラー・要確認処理

| 状況 | 分類 |
|------|------|
| 対象シートが存在しない | **処理停止** |
| 案件ヘッダ行が不正／見つからない | **処理停止**（設定ミス） |
| **10_配車予定**シートが存在しない | **処理停止** |
| 必須ヘッダ列（`運行予定ID`・`予定日` 等）がシートに無い | **処理停止** |
| 担当者行の直下に**車両行がない** | **要確認で続行**（該当ブロックをログ＋スキップ／フラグ） |
| **担当者のみ**／**車両のみ**入力 | **要確認で続行**（§7・§9） |
| **日付が読めない**行 | **要確認で続行**（スキップしログ） |

## 13. 実行方法

### 初期案

- スプレッドシートに**カスタムメニュー**「**配車変換**」を追加し、「**現在のシートを 10_配車予定 へ変換**」を実行する。
- または**図形＋スクリプト割当**で `convertDispatchSheetToSchedule` を紐づける。

### 将来案

- **時間主導トリガ**で毎日実行。
- **配車確定後**の編集イベントにフック（要設計）。
- **変換 → カレンダー同期 → LINE キュー投入**のパイプライン化。
