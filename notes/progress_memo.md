## 2026-04-30（追記）：Claude Code 初回レビュー反映

- **実施**：Claude Code 初回レビュー結果を設計ドキュメントへ反映した。
- **採用した修正内容の要約**：
  - **ID／採番**：内部Keyは **AppSheet UNIQUEID()**、人向けは **表示用番号を別列**（初期は内部Key必須、表示用番号は後続）。
  - **schema_master**：共通ルールに ID・採番を追記。全テーブルに **AppSheet列種別**・**参照先（Ref）** を付与。
  - **配車予定**：ステータス、カレンダーイベントID、最終同期日時、LINE送信済みフラグ、出発地／目的地 を追加。**予定変更は同一運行予定ID維持＋ステータス・更新日時** と明記。
  - **点呼**：点呼種別、アルコール検知値、異常有無、体調確認区分、指示事項、点呼者ID を追加。
  - **運行日報**：**ヘッダ**と**運行明細**に分割。実出発・実到着・便番号・出発地・目的地は **明細側** に配置。
  - **売上**：運行日報ID Ref、請求書番号、入金確認、入金日、請求月 を追加。
  - **LINE通知**：`docs/line_notification_design.md` を新規追加。キュー列と重複防止・GAS でのフラグ更新方針を記載。
- **決定**：採番方式は **内部Key＝UNIQUEID()**、**表示用番号＝別列** で確定。
- **次の作業**：配車予定表の実データ確認に基づき **`schema_master.md` の精密化**（列名確定・必須条件の確定）。
